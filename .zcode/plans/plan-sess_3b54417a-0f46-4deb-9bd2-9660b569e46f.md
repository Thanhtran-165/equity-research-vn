## NEW (this turn) — 4 final additions

### N1. Maturity is an *observed* result, never pre-set (addition §1)
No maturity level is written before execution. Replaced the pre-filled `ROBUST` with:
```yaml
maturity_targets:
  evaluator_engine:        {expected_range: [FUNCTIONAL, ROBUST],                     final_level: null}
  target_verification_layer:{expected_range: [FUNCTIONAL, ROBUST],                     final_level: null}
  target_orchestration:    {expected_range: [EXPERIMENTAL, FUNCTIONAL],                final_level: null}   # NEW bucket
  target_agent_execution:  {expected_range: [EXPERIMENTAL, FUNCTIONAL],                final_level: null}
  overall_target_skill:    {maximum_defensible_level: FUNCTIONAL,                      final_level: null}
```
`target_orchestration` split out (§final-review §5) so Phase 0 isn't forced into either verification or agent buckets. `final_level` filled only from observed evidence at P8. Final report emits `maturity` block with per-tier `{level, evidence{...}, confidence}`.

### N2. Mutations must FAIL for the RIGHT reason (addition §2)
Each mutation is an oracle with a contract, not just "any FAIL":
```yaml
mutation_id: MUT-CAPEX-001
description: "Thay một giá trị CAPEX khỏi ground truth"
mapped_requirement: [REQ-024]
expected_detection: {validator: capex_accuracy_check, result: FAIL}
forbidden_detection: [parser_crash, missing_file, unrelated_schema_failure]
report_must_remain_parseable: true
critical: true
```
A mutation counts as **correctly detected** only when: expected REQ FAILs **AND** expected validator fires **AND** report still parses **AND** no unrelated error. Two KPIs:
```yaml
validator_sensitivity:
  raw_failure_rate: <any FAIL / 5>
  correct_detection_rate: <correctly-detected / 5>   # PRIMARY KPI
  wrong_reason_failure_rate: <forbidden/unrelated FAIL / 5>
  missed_critical_defects: <critical mutations not detected>
```
Hard gate: `correct_detection_rate < 1.0 OR missed_critical_defects > 0 → target_verification_layer capped below ROBUST`.

### N3. Clean-control + specificity (addition §3)
Mutation sensitivity measures only false negatives. Added false-positive axis:
```text
Frozen CLEAN CTD report (copy) → must PASS (clean control)
MUT-001..005 corrupted copies → must FAIL the RIGHT REQ
```
```yaml
validator_specificity:
  clean_controls: 1
  clean_controls_passed: <observed>
  false_positive_count: <observed>
  specificity_rate: <clean_passed / clean_controls>
```
New hard gate **HG-VALIDATOR-FALSE-POSITIVE**: `false_positive_count > 0 → target_verification_layer cannot be ROBUST`. A verifier that FAILs everything has 100% sensitivity but 0% usefulness — specificity catches that.

### N4. Pre-registered, hash-locked protocol (addition §4)
To prevent "moving the goalposts after seeing the score," `evaluation-protocol.lock.yaml` is frozen **before Phase 4 execution**:
```yaml
protocol_version: 0.1.0
frozen_before_execution: true
frozen_at: <iso>
weights: {prs: {sds: 0.25, scs: 0.35, hrs: 0.40},
          sds: {trigger_scope:10, input_contract:15, workflow:20, tool_policy:10, failure:15, validation:15, output:10, maintainability:5},
          hrs: {spec:0.30, workflow:0.20, validation:0.20, output:0.10, recovery:0.10, run_consistency:0.10}}
hrs_publishable_coverage_pct: 70
maturity_thresholds: {stable:{mean_hrs_min:90,minimum_hrs_min:80,pass_rate_min:0.95,std_dev_max:5,critical_failures:0},
                      robust:{golden_edge_pass_rate_min:0.90,autonomous_recovery_rate_min:0.70,critical_failures:0},
                      functional:{golden_pass_rate_min:0.80,critical_failures:0}}
na_policy: excluded_from_denominator
mutation_oracles:
  MUT-CAPEX-001:    {expected_requirement:[REQ-024], expected_validator:capex_accuracy_check}
  MUT-SECTION-001:  {expected_requirement:[REQ-009], expected_validator:section_map_check}
  MUT-ORACLE-001:   {expected_requirement:[REQ-004], expected_validator:artifact_check}
  MUT-VALUATION-001:{expected_requirement:[REQ-025], expected_validator:valuation_recompute_check}
  MUT-DEPLOY-001:   {expected_requirement:[REQ-021], expected_validator:all_requirements_pass}
protocol_sha256: "<generated-after-freeze>"
```
Rule: once P4 starts, the lock is immutable. Any forced change → new protocol version, results reported separately, runs never pooled across protocol versions.
