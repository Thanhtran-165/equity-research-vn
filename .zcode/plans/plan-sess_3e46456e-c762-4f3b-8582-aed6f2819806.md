## Add Peer Feature

### 1. API: `POST /api/public-benchmark/peers` (add to existing route)

Add POST handler to `app/api/public-benchmark/peers/route.ts`:
- Body: `{ name, canonicalUrl, category?, benchmarkRole?, scaleBand?, notes? }`
- Upsert by `canonicalUrl` (unique key)
- Default `benchmarkRole = "watchlist"` (user can promote to `core_peer` later)
- Return created/updated page

### 2. API: `DELETE /api/public-benchmark/peers/[id]` 

New route `app/api/public-benchmark/peers/[id]/route.ts`:
- DELETE handler to remove a peer from benchmark
- Cannot delete own page (guard)
- Returns confirmation

### 3. UI: "Add Peer" panel on `/public-benchmark`

New component `app/public-benchmark/AddPeerPanel.tsx`:
- Collapsible form with: name, Facebook URL, category (dropdown), benchmark role (dropdown), notes
- "Add" button → calls POST API
- Shows existing peers list with remove button
- Glass card pattern matching AutoCollectionPanel

### 4. UI: Update page header actions

Add "Thêm trang" button on `/public-benchmark` page header that toggles the AddPeerPanel.

Fields in the add form:
- **Page name** (required)
- **Facebook URL** (required, must start with facebook.com)
- **Category** (dropdown: stock_market, gold_usd, real_estate, macro_economics, banking_rates, social_commentary, other)
- **Role** (dropdown: core_peer, watchlist, topic_reference)
- **Notes** (optional)
